# Reviewer Checklist

## Scope

- [ ] The package states its review boundary clearly.
- [ ] The package does not overclaim authority.
- [ ] The package distinguishes internal from external evidence.

## Claims and evidence

- [ ] Each major claim appears in the claims register.
- [ ] Each claim has at least one evidence source.
- [ ] Evidence labels match the evidence level.

## Replay

- [ ] Replay steps are explicit.
- [ ] Replay-safe and replay-prohibited actions are distinguished.
- [ ] Replay is not treated as external audit.

## Traceability

- [ ] Claim-to-evidence mapping is present.
- [ ] The traceability matrix is readable and complete.
- [ ] Orphan claims are absent or explained.

## Witnesses

- [ ] Witness limits are stated.
- [ ] Independence is not overstated.
- [ ] Witness attestations are preserved.

## Failures and limits

- [ ] Failed cases are retained.
- [ ] Counterexamples are not removed.
- [ ] Nonclaims are explicit.

## Final judgment

- [ ] Package is consistent.
- [ ] Package is locally supported.
- [ ] Package is not yet externally validated.
