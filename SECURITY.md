# Security Policy

## Supported Versions

Security review currently applies to the active `main` branch.

## Reporting a Vulnerability

Please do not open public issues for sensitive vulnerabilities.

Report privately through GitHub security advisories when available, or contact the repository owner directly.

A useful report includes:

- affected commit or release,
- reproduction steps,
- expected behavior,
- actual behavior,
- impact assessment,
- suggested mitigation, if known.

## Security Boundary

Weaver OS verifies cryptographic authorization claims and release provenance artifacts. It does not independently prove human legitimacy, organizational authorization, physical safety, or deployment safety.

## Promotion Rule

Security-sensitive changes require passing tests and explicit evidence. Narrative claims are not sufficient for authority promotion.
