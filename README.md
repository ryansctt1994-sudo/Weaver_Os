## What This Repository Is

The Weaver Governance Runtime is a **research program** exploring governance architectures for intelligent and adversarial systems.

This repository currently contains a compact verification spine for checking authority claims, replay protection, schema alignment, and release provenance. It is being expanded into a broader evidence-governed research runtime while preserving the original constraint: useful code, tests, schemas, and promotion rules stay; speculative or non-executable material remains non-authoritative until it has implementation evidence.

This repository contains or will contain:

- governance specifications
- experimental implementations
- validation frameworks
- lineage and replay mechanisms
- formal models and proof obligations
- hardware governance concepts
- cryptographic receipts and source manifests

This repository does **not** represent a production platform. All components are research prototypes under active investigation.

## Existing MVP Verification Spine

This repository already includes a compact Weaver verification spine:

```text
triadic_controls/
  crypto/
    replay.py          # Replay-cache protocol and in-memory implementation
    verifier.py        # Authority/refusal signature verifier
  schemas/
    issuer_record.schema.json
    key_registry.schema.json
    role_policy.schema.json
    signature_envelope.schema.json
    verification_result.schema.json

src/weaver_release_guard/
  cli.py               # CLI entry point
  provenance.py        # Release provenance generation and verification
  oidc.py              # Optional OIDC token verification
  utils.py             # Shared hashing, JSON, and encoding helpers

tests/
  triadic_controls/    # Crypto verifier and schema-alignment tests
```

Install:

```bash
python -m pip install -e ".[dev]"
```

Run tests:

```bash
pytest -q
```

Generate release provenance:

```bash
weaver-release-guard generate \
  --dist-dir dist \
  --manifest manifest.json \
  --slsa provenance.json \
  --out weaver_provenance.json \
  --version 0.5.0 \
  --build-id "$BUILD_ID" \
  --policy-version v0.5.1 \
  --lease-id "$LEASE_ID" \
  --authority-level 3
```

Verify release provenance:

```bash
weaver-release-guard verify \
  dist/example.whl \
  manifest.json \
  weaver_provenance.json \
  --slsa provenance.json
```
