# Daily Operator Loop

## Objective

Keep project continuity current without allowing unreviewed AI output to mutate canon.

## Procedure

1. Collect new source material.
2. Run Weaver Sync.
3. Run Evidence Audit on new claims.
4. Write outputs to `OUTPUTS/`.
5. Mark each item as preserved, rejected, implementation-target, or promotion-candidate.
6. Run Receipt Check only for promotion candidates.
7. Update registry only after review.

## Output checklist

- Source summarized.
- Claims extracted.
- Artifacts listed.
- Evidence labels applied.
- Risks identified.
- Next actions specified.
- Promotion boundary preserved.

## Success criteria

The project state becomes easier to reconstruct, and no unsupported claim gains authority.
