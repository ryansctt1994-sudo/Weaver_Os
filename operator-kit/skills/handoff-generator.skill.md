# Skill: Handoff Generator

## Purpose

Produce a compact professional handoff package that allows a new AI session, engineer, or reviewer to reconstruct the current project state without inheriting unsupported authority.

## Procedure

1. State the project objective.
2. List the current freeze state or known canonical baseline.
3. Summarize verified artifacts and preservation-only artifacts separately.
4. Capture active decisions, unresolved risks, and next actions.
5. Include evidence labels and authority boundaries.
6. Exclude unnecessary narrative.

## Output sections

1. Executive state.
2. Canonical invariants.
3. Load-bearing artifacts.
4. Preservation-only artifacts.
5. Current risks.
6. Open validation tasks.
7. Immediate next actions.
8. Non-goals.

## Quality bar

A valid handoff should let a fresh session recover the operating frame, current milestone, evidence ladder, authority boundaries, agent promotion rules, active objectives, and known non-goals.
