# E3.5 Reproduction Kit Specification

**Status:** Canonical Gate Artifact  
**Purpose:** Independent reproduction without interpretation drift  
**Authority:** Required before E3.5 promotion

## Purpose

This specification enables an independent operator to rebuild the Cathedral OS governance environment, replay the canonical `authority_ledger.json`, compute the canonical `STATE_DIGEST`, and attest to the result without founder intervention.

## Prerequisites

- Docker 20.10+ and Docker Compose 2.0+
- Python 3.9+ for verifier execution
- Go 1.23+ if rebuilding the server

## 1. Environment Pinning

- Build the Docker images from the provided `git_commit`.
- Ensure `published_manifest.json` matches the commit hash and canonical hashes.
- Treat `published_manifest.json` and `authority_ledger.json` as the single source of truth for E3.5 reproduction.

## 2. Replay Procedure

1. Initialize the `chronicle-raft` cluster with three nodes.
2. Submit the events from `authority_ledger.json` to the leader via `POST /events`.
3. Let the cluster commit the events.
4. Wait for the cluster to reach consensus.
5. Check `/head` to confirm the canonical head.

## 3. Verification Protocol: `make verify-e35`

The provided `verify-e35` Makefile target performs the following:

1. Confirms `published_manifest.json` and `authority_ledger.json` are present.
2. Rebuilds the independent environment.
3. Injects `authority_ledger.json` through the API where a cluster is available.
4. Computes:
   - `HEAD_HASH`
   - `STATE_DIGEST`
   - `AUTHORITY_LEDGER_HASH`
   - `MANIFEST_HASH`
5. Compares computed values against `published_manifest.json`.

## 4. Failure Conditions

- **Exit Code 0:** Reproduction successful. `HEAD_HASH`, `STATE_DIGEST`, `AUTHORITY_LEDGER_HASH`, and `MANIFEST_HASH` match published values.
- **Exit Code 1:** Any hash mismatch.
- **Exit Code 2:** Missing artifact or invalid artifact shape.
- **Exit Code 3:** Attestation signature verification failed.

## 5. Witness Rule

E3.5 is not crossed by local author execution. E3.5 is crossed only when an independent operator runs the reproduction kit, obtains a successful result, and archives the signed witness attestation in `evidence_bundle/`.

## 6. Promotion Boundary

Promotion beyond E3.5 requires independent evidence. E4 requires two-of-three signed independent witness reproductions. Local success remains useful engineering evidence, but it is not independent reproduction evidence.
